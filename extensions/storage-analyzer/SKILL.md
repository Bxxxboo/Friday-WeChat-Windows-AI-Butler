# Storage Analyzer — 全盘扫描技能

## 铁律
- **全盘覆盖**：必须对 C/D/E 三盘（或系统所有可用磁盘）逐一详细分析，禁止一带而过
- **只读扫描**：全程只调用 get_disk_usage、find_duplicates、get_system_status、get_top_processes、get_network_info 等只读工具
- **审批删除**：任何删除/清理操作必须列出明细并获得用户确认后才能执行
- **三盘均详**：每个磁盘必须有独立的分析板块，包含使用率、风险等级、主要占用来源、清理建议
- **交互报告**：每次分析必须生成交互式 HTML 报告（可点击展开建议），并在浏览器中打开

## 执行流程

### 1. 数据采集（并行调用）
```
get_disk_usage                          → 所有磁盘使用率
get_system_status                       → CPU / 内存 / 开机时间
get_top_processes(limit=15)             → 内存大户
get_network_info                        → 主机名 / IP
find_duplicates(max_files=300, path=<工作区>)  → 重复文件
```

### 2. 逐盘分析
对每个磁盘，必须输出以下结构化信息：
| 字段 | 来源 |
|------|------|
| 使用率百分比 | get_disk_usage |
| 总容量 / 已用 / 剩余 | get_disk_usage |
| 风险等级（🟢<60% 🟡60-80% 🔴>80%） | 计算 |
| 距 80% 警戒线缓冲 | 计算 |
| 距 90% 危险线缓冲 | 计算 |
| 主要占用来源（目录级） | list_directory / 推测 |
| 专属清理建议 | 根据使用率生成 |

### 3. 分级建议
- 🔴 高优先级：使用率 >70% 或缓冲 <30GB → 大文件扫描 + 迁移方案
- 🟡 中优先级：重复文件清理、临时文件
- 🟢 低优先级：后台进程优化、缓存清理（空间充裕时）

### 4. 生成 HTML 报告
按 `{plugin_dir}/templates/report.html` 模板结构，填入实际数据后写入 `storage-report.html`，用 `open_app` 在浏览器打开。

### 5. 用户交互
报告打开后，对每条建议提供可展开的详细操作步骤。用户点击建议后，根据建议内容执行对应的清理操作（需审批）。
