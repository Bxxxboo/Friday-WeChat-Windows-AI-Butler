# 文件删改安全准则

**目标**：防止误删、误覆盖用户重要资料。星期五的审批弹窗是最后防线，不得绕过。

---

## 1. 铁律（违反即错误）

1. **删、改、移动用户电脑上已存在的文件/文件夹** → 必须使用对应工具（`delete_file`、`delete_directory`、`write_text_file`、`move_file`、`organize_directory`、`batch_rename` 等），并**等待用户在界面/微信中点击「同意」**后再执行。
2. **禁止**用 `run_python` / `run_python_script` / `run_powershell` 执行 `os.remove`、`unlink`、`shutil.rmtree`、`shutil.move`、`os.replace` 等**删除/覆盖已有文件**的操作来**替代**专用工具或**绕过**审批。工作区内**新建**文件可经普通 `run_python`（同轮确认一次）；**覆盖已有同名文件**仍须 `write_text_file` 等专用工具并审批。
3. **禁止**未经用户明确同意删除、覆盖、移动任何文件；「帮我整理」「清理一下」「修复锁定」**不等于**同意删除。
4. **禁止**修改 `%AppData%\\Friday\\` 下星期五自身配置（`settings.json`、`operations.json` 等）；应用问题应说明步骤让用户在设置里处理，或只读排查。

---

## 2. 操作前必须做的

| 场景 | 要求 |
|------|------|
| 删除 / 清空 | 先只读确认目标；说明**路径、数量、是否可恢复**；等审批 |
| 覆盖写入 | 说明是否覆盖已有文件；等 `write_text_file` 审批 |
| 批量整理 | 先只读扫描 → 给出**计划清单** → 用户确认后再移动/整理 |
| 重复文件 | 默认**只列出**；删除须用户逐批确认 |
| 修复/清理 | 先诊断只读；**不得**擅自删临时文件、重写 JSON |

---

## 3. 工具选择

- ✅ 删改移 → 专用文件工具 + 审批
- ❌ Python/PowerShell 脚本删写用户文件
- ❌ 一次同意后默认可删其他路径

---

## 4. 敏感路径（默认只读）

桌面、文档、下载、AppData、系统目录、办公文档、项目源码、备份、`%AppData%\\Friday\\` — 须用户逐路径确认后才可删改。

---

## 5. 自检（写/删工具前）

- [ ] 只读确认路径？
- [ ] 说明路径、操作、可恢复性？
- [ ] 专用工具 + 等审批？
- [ ] 未用 Python 绕过？

未获审批即删改 = 失败；收尾须列出实际动过的路径，或明确「未删除任何文件」。
