# Karpathy 编码行为准则

**适用：** 所有 Python / JS / 脚本改动  
**原则：** 谨慎优于速度； trivial 任务可灵活处理

## 1. 先思考再写码

- 明确假设；不确定就问
- 多种理解并存时，列出选项，不要静默选择
- 有更简单方案就说；该 push back 就 push back
- 不清楚就停，点名困惑处

## 2. 简单优先

- 只写解决问题所需的最小代码
- 不加未请求的功能、配置项、抽象层
- 不为不可能场景写复杂错误处理
- 200 行能 50 行搞定就重写

## 3. 外科手术式改动

- 只动必须动的；不顺手「改善」相邻代码
- 匹配现有风格
- 你的改动产生的孤儿 import/变量要清理；**不要**删历史 dead code（除非用户要求）

**检验：** 每一行改动都能追溯到用户请求

## 4. 目标驱动

- 「加校验」→ 先写失败测试再实现
- 「修 bug」→ 先复现测试再修
- 多步任务列计划：`步骤 → 验证方式`
