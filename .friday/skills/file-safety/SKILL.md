# 文件删改安全准则

**目标**：防止误删、误覆盖用户重要资料。星期五的审批弹窗是最后防线，不得绕过。

与 [karpathy-guidelines](../karpathy-guidelines/SKILL.md)（最小改动）和 [surgical-bugfix](../surgical-bugfix/SKILL.md)（修 bug 范围）配合使用。

---

## 1. 铁律（违反即错误）

1. **删、改、移动用户电脑上已存在的文件/文件夹** → 必须使用对应工具（`delete_file`、`delete_directory`、`write_text_file`、`move_file`、`organize_directory`、`batch_rename` 等），并**等待用户在界面/微信中点击「同意」**后再执行。
2. **禁止**用 `run_python` / `run_python_script` / `run_powershell` 执行 `os.remove`、`unlink`、`shutil.rmtree`、`shutil.move`、`os.replace` 等**删除/覆盖已有文件**的操作来**替代**专用工具或**绕过**审批。工作区内**新建**文件可经普通 `run_python`（同轮确认一次）；**覆盖已有同名文件**仍须 `write_text_file` 等专用工具并审批。
3. **禁止**未经用户明确同意删除、覆盖、移动任何文件；「帮我整理」「清理一下」「修复锁定」**不等于**同意删除。
4. **禁止**修改 `%AppData%\\Friday\\` 下星期五自身配置（`settings.json`、`operations.json` 等）；应用问题应说明步骤让用户在设置里处理，或只读排查。

---

## 2. 操作前必须做的

| 场景 | 要求 |
|------|------|
| 删除 / 清空 | 先 `list_directory` 或 `read_text_file` 确认目标；说明**路径、数量、是否可恢复**；等审批 |
| 覆盖写入 | 说明是否覆盖已有文件；等 `write_text_file` 审批（界面会展示 diff） |
| 批量整理 | 先只读扫描 → 给出**计划清单** → 用户确认后再 `move_file` / `organize_directory` |
| 重复文件 | 默认**只列出**；删除须用户逐批确认 |
| 修复/清理类任务 | 先诊断只读；**不得**擅自删临时文件、重写 JSON，除非用户明确点名路径并同意 |

---

## 3. 工具选择（禁止走捷径）

```
✅ 删除单个文件     → delete_file
✅ 删除文件夹       → delete_directory
✅ 新建或覆盖文本   → write_text_file（覆盖会触发审批）
✅ 移动/重命名      → move_file / batch_rename
✅ 批量整理         → organize_directory

❌ run_python 里 os.remove / shutil.rmtree / shutil.move / 重写 operations.json
✅ run_python 在工作区新建 report.py、out.json 等（同轮确认一次；勿用来覆盖已有文件）
❌ run_powershell Remove-Item 删用户资料（除用户明确要求且已审批外）
❌ 同一对话里「已点过一次同意」就默认可删其他路径（每条危险操作仍须审批）
```

---

## 4. 敏感路径（默认只读）

未经用户**逐路径**书面/口头确认，不得删改：

- 桌面、文档、下载、图片、视频及用户指定的工作目录
- `%AppData%`、`Program Files`、系统盘根目录
- 任何 `.docx` / `.xlsx` / `.pdf` / 项目源码 / 数据库 / 备份文件
- 星期五目录 `%AppData%\\Friday\\`

**工作区内新建**文件（用户任务明确要「生成报告/脚本」且路径在工作区）可写入，但**覆盖已有同名文件**仍须审批。

---

## 5. 对话模式差异

| 模式 | 删改行为 |
|------|----------|
| **Ask** | 只读；仅建议，不调用写/删/执行工具 |
| **Agent** | 每次删/写/执行均需用户确认（除非设置关闭，仍建议说明后果） |
| **Yolo** | 工作区内部分操作可自动；**删除、PowerShell、Python 执行**仍须确认 |

微信远程：审批在微信回复「同意/拒绝」；不得因远程而跳过说明。

---

## 6. 推荐话术（向用户）

执行前：

> 我准备删除/覆盖 `[路径]`（约 N 个文件）。删除后通常难以恢复。请在星期五弹窗点「同意」后继续。

只读阶段：

> 我先扫描列出结果，不会直接删除；确认清单后您再决定是否清理。

---

## 7. 自检清单（调用写/删工具前）

- [ ] 是否已用只读工具确认目标存在且路径正确？
- [ ] 是否向用户说明了**路径 + 操作类型 + 是否可恢复**？
- [ ] 是否使用专用工具而非 Python/PowerShell 删写？
- [ ] 是否等待审批通过（未假设「用户说整理 = 同意删除」）？
- [ ] 是否避开了 AppData/星期五配置与系统目录？

---

## 8. 完成定义

- 未获审批即删改 = **失败**，须向用户道歉并说明如何恢复（回收站/备份）。
- 收尾回复须列出**实际动过的路径**；若仅只读扫描，明确写「未删除任何文件」。
